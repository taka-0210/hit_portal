# Design Concept

Version 1.0

---

# Purpose

このポータルは、社内業務を支えるための情報基盤である。

デザインの目的は「美しさ」ではない。

社員が毎日ストレスなく利用でき、
改善活動が自然に続く環境をつくることを目的とする。

---

# Design Philosophy

私たちは、派手な管理画面を作らない。

目指すのは、

**毎日開きたくなる仕事のホーム画面**である。

デザインは目立つためではなく、

仕事をしやすくするために存在する。

---

# Design Principles

## 1. Comfortable

長時間利用しても疲れない。

情報量は多くても圧迫感を与えない。

余白を大切にする。

---

## 2. Simple

迷わせない。

余計な装飾は行わない。

必要な情報だけを分かりやすく表示する。

---

## 3. Organized

情報には必ず整理された居場所がある。

利用者は考えなくても目的の情報へたどり着ける。

---

## 4. Professional

業務システムとして信頼できる品質を保つ。

落ち着きがあり、

安心して毎日利用できるデザインとする。

---

## 5. Warm

無機質な管理画面にはしない。

人が使うシステムとして、

適度な温かみを持たせる。

---

# Dashboard Concept

Dashboard は管理画面ではない。

社員が一日の仕事を始める場所である。

そのため、

今日必要な情報だけを表示する。

例

- 通知
- 承認待ち
- 今日の予定
- 最近更新された情報
- 進行中の改善
- よく使う機能

---

# Layout

画面構成はシンプルに統一する。

```
┌──────────────────────────────┐
│ Header                       │
├──────────┬───────────────────┤
│ Sidebar  │                   │
│          │ Main Content       │
│          │                   │
│          │                   │
└──────────┴───────────────────┘
```

---

# Color Policy

全体は白を基調とする。

アクセントカラーのみブランドカラーを使用する。

背景は柔らかいグレーを採用し、

目への負担を軽減する。

---

# Typography

読みやすさを最優先する。

装飾的なフォントは使用しない。

見出し・本文・補足情報の階層を明確にする。

---

# Components

すべての画面で統一された UI を使用する。

- Card
- Button
- Table
- Form
- Badge
- Modal
- Search
- Pagination

同じ役割の UI は必ず同じデザインとする。

---

# Icons

線画アイコンを採用する。

過度なイラストや装飾は使用しない。

アイコンは補助情報として利用する。

---

# Motion

アニメーションは最小限とする。

画面遷移やホバー時に、

自然で短い動きを付ける。

派手な演出は行わない。

---

# Mobile

PC を基本としつつ、

営業・現場でも利用できるよう

スマートフォン対応を前提とする。

---

# Accessibility

誰でも迷わず操作できることを優先する。

色だけで情報を伝えない。

文字サイズやコントラストにも配慮する。

---

# Design Keywords

- Clean
- Simple
- Comfortable
- Organized
- Reliable
- Professional
- Warm

---

# Non Goals

このデザインは、

・派手さを競うものではない。

・最新トレンドを追うことを目的としない。

・装飾を増やすことを目的としない。

・管理画面らしい無機質な UI を目指さない。

---

# Final Message

デザインは仕事を支える道具である。

社員が毎日使いたくなり、

改善を積み重ねたくなる。

そんな業務改善プラットフォームを目指す。

画面を作るのではない。

仕事がしやすい環境をデザインする。
